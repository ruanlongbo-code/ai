# 需求说明书

# 项目概述

# 1.1 项目背景

本系统为一套线上商城系统，支持用户注册、登录、浏览商品、购物车管理、订单生成与支付、评论等功能，旨在为用户提供完整的电商购物闭环体验。

# 1.2 项目目标

开发一套 B2C 商城平台，支持商品展示、购物流程、订单管理、用户信息管理等核心模块，并具备完善的接口设计，便于前后端分离开发。

# 1.3 用户角色

普通用户：浏览商品、下单、支付、查看订单、评论等管理后台用户（此版本暂未涉及）：可拓展为商品管理、订单管理、用户管理等后台操作

# 一、用户模块

# 模块介绍

为用户提供从注册、登录、资料维护、收货信息管理等基本账户管理功能，构建系统的“用户中心”基础。

# 适用角色

普通用户（注册用户）未登录用户（注册/登录）

# ?? F1.1 用户注册

# ?? 功能背景

新用户通过注册方式创建账户，支持邮箱/用户名 $^ +$ 密码的注册方式。

# ?? 主流程

1. 用户打开注册页，填写注册信息  
2. 系统校验格式与唯一性（用户名、邮箱）  
3. 提交注册，后台创建账户，初始状态为“正常”  
4. 注册成功后自动登录并跳转首页

# ⚠ 异常流程

邮箱/用户名已被注册：提示“已存在”两次密码不一致：提示用户重新输入

# ?? 状态规则

新用户状态为 “正常”注册时间记录为创建时间，头像为默认图

# ?? 业务规则

用户名唯一，支持 4\~20 位字母数字组合密码长度不少于 6 位邮箱必须符合格式 xxx@xxx.xx

# ?? F1.2 用户登录

# ?? 功能背景

用户通过用户名/邮箱 $^ +$ 密码登录系统。

# ?? 主流程

1. 用户输入账户 $^ +$ 密码提交  
2. 系统验证账号是否存在、密码是否正确  
3. 验证通过 登录成功，跳转首页  
4. 登录失败 显示错误提示

# ⚠ 异常流程

密码错误：提示“账号或密码错误”连续失败超过 5 次：临时锁定 10 分钟（防爆破）

# ?? 状态规则

登录成功后刷新用户活跃时间登录设备、时间、IP 记录入日志

# ?? 业务规则

账号存在性通过用户名/邮箱模糊判断同时支持邮箱登录和用户名登录

# ?? F1.3 用户信息修改

?? 功能背景

用户可修改昵称、密码、头像、性别等基础信息。

# ?? 主流程

1. 用户进入“个人中心”  
2. 修改某字段并保存  
3. 系统校验内容合法性（如昵称长度、头像格式）  
4. 修改成功后刷新显示

# ⚠ 异常流程

用户未登录：提示登录后操作输入非法字符：提示不符合规范

# ?? 业务规则

昵称长度 $\leq 2 0$ ，支持中英文  
性别只能为“男 / 女 / 保密”  
头像图片限制大小（2M以内），格式为 png/jpg/jpeg

# ?? F1.4 手机号/邮箱绑定与解绑

# ?? 功能背景

为提升账号安全性和通知能力，用户需绑定手机号或邮箱。

# ?? 主流程

1. 用户申请绑定/更换手机号或邮箱  
2. 系统发送验证码  
3. 用户输入验证码验证成功  
4. 修改绑定信息并提示成功

# ⚠ 异常流程

验证码超时或错误：提示“验证码无效”新邮箱/手机号已被绑定：提示“该账号已存在绑定关系”

# ?? 业务规则

验证码 5 分钟有效每小时最多发送 5 次，防刷机制同一邮箱/手机号只能绑定一个账号

# ?? F1.5 收货地址管理

# ?? 功能背景

用户可新增、编辑、删除收货地址，并设置默认地址用于下单。

# ?? 主流程

1. 用户进入“我的地址”管理页  
2. 新增/编辑地址，包括姓名、电话、区域、详细地址等  
3. 系统校验字段格式  
4. 保存成功后列表展示更新

# ⚠ 异常流程

电话号格式不符、字段为空 拒绝提交地址超限（最多5个） 提示不能添加更多地址

# ?? 状态规则

新增第一个地址自动设为默认删除默认地址后，自动设下一条为默认

# ?? 业务规则

每个用户最多绑定5个地址每个地址可被设置为“默认地址”状态唯一

# ?? F1.6 获取验证码

# ?? 功能背景

验证码用于绑定账号、身份验证、找回密码等关键环节。

# ?? 主流程

1. 用户点击“获取验证码”  
2. 系统验证发送频率与目的合法性  
3. 向目标手机号/邮箱发送验证码  
4. 用户输入进行验证

# ⚠ 异常流程

发送过快：提示“请勿频繁操作”验证码已失效：提示“请重新获取”

# ?? 业务规则

同一手机号/邮箱每分钟限发1次，每小时最多5次  
验证码为6位数字，有效期5分钟  
验证成功后销毁验证码记录

# ?? F2.1 首页商品展示

# ?? 功能背景

用户进入商城首页，需看到最具吸引力的商品及分类，提升用户停留率和转化。

# ?? 主流程

1. 用户打开首页  
2. 系统依次展示轮播图、分类列表、推荐商品（如热门、折扣等）

# ?? 业务规则

轮播图内容由后台配置  
分类展示最多显示6个主分类  
推荐商品优先按销量或评分排序，可缓存5分钟

# ?? F2.2 商品列表浏览

# ?? 功能背景

支持用户按商品分类、价格区间、销量、人气等维度进行筛选、排序与分页浏览。

# ?? 主流程

1. 用户点击某分类或关键词进入列表页  
2. 页面支持选择排序方式（默认、价格升/降、销量等）  
3. 支持分页浏览，显示商品封面、标题、价格等

# ⚠ 异常流程

筛选无结果：显示“暂无匹配商品”排序参数非法：返回默认排序

# ?? 业务规则

每页默认显示 20 条记录  
分类ID需有效  
排序字段仅支持白名单字段（防SQL注入）

# ?? F2.3 商品详情页

# ?? 功能背景

提供商品完整信息，包括图片、价格、库存、参数、用户评论、售后说明等，帮助用户做出购买决策。

# ?? 主流程

1. 用户点击某商品进入详情页  
2. 展示商品主图、价格、介绍、参数说明  
3. 支持切换详情图文 / 用户评论标签页  
4. 显示“加入购物车”或“立即购买”按钮

# ⚠ 异常流程

商品不存在：返回 404 提示商品已下架：显示“商品已下架，无法购买”

# ?? 业务规则

商品主图最多展示 5 张，支持左右滑动评论展示前 10 条，可分页查看售后/退换货说明固定显示在底部区域

# ?? F2.4 商品收藏 / 取消收藏

# ?? 功能背景

用户可收藏感兴趣的商品，便于后续查看与复购。

# ?? 主流程

1. 用户点击“收藏”按钮  
2. 若已登录，系统记录用户与商品的收藏关系  
3. 再次点击为“取消收藏”

# ⚠ 异常流程

未登录用户点击收藏：提示登录商品ID非法：忽略操作并提示

# ?? 状态规则

商品收藏状态可实时反映在列表/详情页中收藏记录存在数据库中，可用于“我的收藏”列表

# ?? 业务规则

同一用户收藏同一商品仅记录一次支持取消收藏操作用户可最多收藏 500 个商品（上限可后台配置）

# ?? F3.1 加入购物车

# ?? 功能背景

用户在浏览商品时，若未准备立即购买，可先将商品加入购物车暂存，方便批量结算。

# ?? 主流程

1. 用户点击“加入购物车”  
2. 系统判断用户是否已登录  
3. 若该商品已存在于购物车中 增加数量；否则新增记录  
4. 提示“加入成功”，不跳转

# ⚠ 异常流程

商品不存在或已下架 提示“该商品不可购买”用户未登录 提示跳转登录页

# ?? 业务规则

每个用户购物车上限为 100 条记录默认选中状态为 true商品数量上限为库存值，下限为 1

# ?? F3.2 修改商品数量 / 状态

# ?? 功能背景

支持用户在购物车中调整每项商品的购买数量，以及是否参与结算（选中状态）。

# ?? 主流程

1. 用户点击 $" + / \mathrm { ~ - ~ } "$ 调整数量，或点击选中/取消  
2. 系统实时保存修改后的状态  
3. 数量为 0 时提示是否删除该商品

# ⚠ 异常流程

数量超过库存 提示“超过最大库存”数量为负数 拒绝并提示“数量不能小于1”商品已下架 自动设为不可选中并提醒用户

# ?? 业务规则

数量支持 1\~库存最大值，超出则限制选中状态参与结算，取消选中不参与提交订单

# ?? F3.3 删除购物车商品

# ?? 功能背景

用户可删除购物车中已失效或不再需要购买的商品，支持单个或批量删除操作。

# ?? 主流程

1. 用户在购物车界面选择商品  
2. 点击“删除”按钮后确认  
3. 系统从购物车记录中移除对应项，刷新页面

# ⚠ 异常流程

商品ID非法 忽略处理并提示已删除项再次操作 提示“商品已不存在”

# ?? 业务规则

删除操作仅影响当前用户，不影响商品库存或状态删除后商品不再参与结算，UI立即刷新

# ?? A6.1 轮播图管理

# ?? 功能背景

轮播图位于首页顶部，是用户进入页面后的第一视觉焦点，常用于推广活动、热卖商品、专题页引导。

# ?? 轮播图字段建议：

<table><tr><td>字段</td><td>类型</td><td>说明</td></tr><tr><td>图片地址</td><td>image_url</td><td>支持CDN加速</td></tr><tr><td>标题</td><td>string</td><td>可选，默认不展示</td></tr><tr><td>跳转链接</td><td>url</td><td>点击跳转的活动页／商品页</td></tr><tr><td>状态</td><td>枚举</td><td>上架/下架</td></tr><tr><td>排序权重</td><td>number</td><td>越小越靠前</td></tr><tr><td>展示时间段</td><td>时间范围</td><td>开始时间~结束时间</td></tr></table>

# ?? 业务规则：

首页最多展示 5 张轮播图  
● 已过期轮播图不展示  
● 图片建议尺寸统一，如 $7 5 0 { \times } 3 0 0 { \ p } { \times }$   
所有跳转链接需合法校验（避免空链接或死链）

# ?? A6.2 推荐商品位配置

# ?? 功能背景

用于配置首页推荐商品、分类推荐商品、搜索推荐区等，提升商品曝光和转化。

?? 配置字段建议：  

<table><tr><td>字段</td><td>说明</td></tr><tr><td>推荐位类型</td><td>如：首页顶部、首页中部、分类推荐等</td></tr><tr><td>商品ID</td><td>绑定的商品唯一编号</td></tr><tr><td>显示名称</td><td>默认使用商品标题</td></tr><tr><td>是否固定展示</td><td>可设置为“锁定展示&quot;或“随排序变化”</td></tr><tr><td>排序</td><td>控制推荐位顺序</td></tr></table>

# ?? 业务规则：

每个推荐位最多支持展示 10 个商品推荐商品不可绑定“下架商品”，需校验商品状态推荐商品支持按“活动时间”控制展示周期

# ?? A6.3 首页公告栏配置

# ?? 功能背景

配置首页顶部或中部公告内容，用于展示节日通知、发货延迟、隐私更新等系统消息。

# ?? 字段建议：

<table><tr><td>字段</td><td>示例</td></tr><tr><td>公告内容</td><td>“五一假期发货通知：5月1日暂停发货&quot;</td></tr><tr><td>发布时间</td><td>yyyy-MM-dd HH:mm:ss</td></tr><tr><td>状态</td><td>展示/隐藏</td></tr><tr><td>权重</td><td>多条公告控制展示顺序</td></tr></table>

# ?? 业务规则：

支持多个公告位并行显示  
支持“定时生效”与“定时下线”  
文本支持超链接或简单富文本格式（如跳转详情页）

# ?? A6.4 推荐内容上下架控制

# ?? 功能背景

内容可随运营活动实时上架/下架，控制展示周期，灵活适配短促营销。

?? 状态说明：  

<table><tr><td>状态</td><td>含义</td></tr><tr><td>上架</td><td>页面可见，立即生效</td></tr><tr><td>下架</td><td>页面不可见，数据保留</td></tr><tr><td>定时上架</td><td>达到指定开始时间后自动生效</td></tr><tr><td>定时下架</td><td>到达失效时间后自动隐藏</td></tr></table>

# ?? 应用范围：

可应用于轮播图 / 商品推荐位 / 公告栏等所有“内容型组件”
