# 用户管理系统需求文档（示例PRD）

## 1. 概述
本系统为企业内部用户管理平台，支持用户注册、登录、信息管理、角色权限控制等核心功能。

## 2. 功能模块

### 2.1 用户注册
- 用户通过手机号 + 验证码注册。
- 手机号格式校验：11位数字，以1开头。
- 验证码有效期为5分钟，6位纯数字。
- 同一手机号60秒内只能发送一次验证码。
- 注册成功后自动登录并跳转首页。
- 已注册手机号不允许重复注册，需提示"该手机号已注册"。

### 2.2 用户登录
- 支持手机号 + 密码登录。
- 支持手机号 + 验证码登录。
- 连续5次密码错误，账号锁定30分钟。
- 登录成功后生成 JWT Token，有效期24小时。
- Token 过期后需重新登录。

### 2.3 用户信息管理
- 用户可修改昵称、头像、邮箱。
- 昵称长度限制：2-20个字符。
- 头像支持 jpg/png 格式，大小不超过 2MB。
- 邮箱格式需符合标准邮箱格式。
- 修改信息需验证当前密码。

### 2.4 角色权限
- 系统角色：超级管理员、管理员、普通用户。
- 超级管理员可管理所有用户和角色。
- 管理员可管理普通用户，不可管理其他管理员。
- 普通用户仅可查看和修改自己的信息。
- 角色变更需超级管理员审批。

### 2.5 用户列表与搜索
- 管理员可查看用户列表，支持分页（每页20条）。
- 支持按手机号、昵称、注册时间模糊搜索。
- 支持按角色筛选。
- 支持批量禁用/启用用户。
- 导出用户列表为 Excel（最多导出5000条）。

## 3. 非功能性需求
- 接口响应时间 ≤ 200ms（P99）。
- 支持 1000 并发用户。
- 用户密码使用 bcrypt 加密存储。
- 所有敏感操作记录审计日志。
- 系统可用性 ≥ 99.9%。
